UDP 模拟可靠传输系统（带三次握手 + 丢包模拟）- 运行说明文档

项目简介
本项目实现了一个简化版的 UDP 可靠数据传输机制，包括客户端与服务端基于 UDP 的三次握手连接建立，滑动窗口控制，包序号管理，服务端模拟丢包，客户端自动重传与 RTT 估算，最终统计丢包率、RTT 最大/最小/平均值和标准差

文件结构说明
-"udpclient.py"：UDP 客户端程序，发送数据包、收 ACK、统计性能
-"udpserver.py"：UDP 服务端程序，建立连接、模拟丢包、ACK 应答
-"readme.txt"：运行说明文档

运行环境
Python 3.6 及以上版本
第三方依赖库：
"pandas"（用于数据统计）

使用方法

1. 启动服务器端
python udpserver.py
运行后会监听本地 2601 端口，等待客户端连接。
2.启动客户端
python udpclient.py ip portnumber
ip (IP地址，如：127.0.0.1)
portnumber(端口号，默认：2601)

客户端会启动三次握手流程建立连接，向服务器端发送编号为1-30的数据包（每包80字节），接受ACK并计算RTT，丢包时自动重传

通信协议说明
连接建立（三次握手）：
客户端发送：SYN
服务端响应：SYN-ACK
客户端确认：ACK
数据包格式：
前 4 字节为包编号（大端序）
第 5~6 字节为标志位 0x06 0x01
第 7~80 字节为正文内容
ACK 包格式：
前 4 字节为确认号
后 2 字节为标志 0x06 0x01

示例
（客户端）
发送 SYN 请求，等待服务端确认...
收到服务端 SYN-ACK
发送 ACK，连接建立完成
发送包 1: 内容大小 80 字节
收到 ACK 1, RTT: 22.35 ms
...
所有数据包发送完成并收到确认，重传次数：2 次
丢包率: 6.67%
最大 RTT: 45.81 ms
最小 RTT: 12.21 ms
平均 RTT: 24.33 ms
RTT 标准差: 9.52 ms
